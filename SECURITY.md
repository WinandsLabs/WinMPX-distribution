# Security Policy

## Supported Versions
We release patches for security vulnerabilities in the following versions:

| Version | Supported          |
| ------- | ------------------ |
| latest  | ✅                 |
| older   | ❌                 |

## Reporting a Vulnerability
If you discover a security vulnerability in this project, please help us by reporting it responsibly.

- **Open a public issue.**
- Or send an email to: [support@winmpx.com]  
- Include as much detail as possible (steps to reproduce, affected version, potential impact).

We will:
1. Acknowledge receipt of your report within 48 hours.
2. Investigate and confirm the issue.
3. Provide a fix or mitigation plan.
4. Credit you in the release notes if you wish.

## Security Best Practices for Users
- Always use the latest release.
- Do not run binaries from untrusted sources.
- Verify checksums/signatures when available.

## Disclosure Policy
We follow a **responsible disclosure** model:
- Vulnerabilities are fixed before public disclosure.
- Critical issues may be announced with limited details until a patch is available.
